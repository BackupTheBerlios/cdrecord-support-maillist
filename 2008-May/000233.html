Nearly 100% of today's software depends on "confidure" and thus depends on 
the ability to run tests _on_ the target platform. It is not possible to 
cross-compile such software unless you _manually_ create _all_ expected
auto-configuration results for the target platform.

>             While cross compiling the latest source it will show error
> message like (configure: error: cannot run test program while cross
> compiling).
>
>             So I cannot able to cross compile.

See above: no recent software allows cross-compilation unless you manually
create the auto-configuration results or fetch them from a native compilation
run on a machine of the same architecture and sub-architecture.
