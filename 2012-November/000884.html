Hello Jörg (and others),

I would maintain and include (at least) smake and cdrtools in Debian
and Ubuntu Linux distributions. I'm concerned about the exception
clauses to both CDDL and GPL in the cdrtools 3.00 source distribution,
and multiple licenses or restrictions within the code and/or build
system.

I.e. "Begin restricted code for quality assurance."  (cdrecord/cdrecord.c:362)

I have read that debian-legal rejected the instruction that code
(having to do with the build system detecting modified code or build
system, in this example) is not allowed to be modified;  it is thus a
restriction of freedom and so does not follow the Debian Free Software
Guidelines.

The instruction about ""not being allowed to modify code"" and
"restricted code" sections is the basis on which cdrtools 2.xx was
removed from the Debian archives. Is there an acceptable,
DFSG-compatible way to let the user know an appropriate venue for
technical support?

Possible solutions?

Kind regards,

Eric Shattow
