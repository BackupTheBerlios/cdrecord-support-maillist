version: 2.01.01a44

My problem: It is hard to understand the result (dev=bus/target/lun)
for entry level user (especially USB/Firewire CD-RW drive has nothing
to do with SCSI). If cdrecord supports "dev=E:", it would be
very convenient for entry level user!

I know cdrecord supports "dev=/dev/hda" in Linux, which is
very convenient for entry level user to avoid inputing SCSI
"dev=bus/target/lun", especially USB/Firewire CD-RW drive has nothing
to do with SCSI.

BTW, if you have two or more CD-RW drives with the same hardware
(e.g., vendor/revision), it is hard to differentiate them thru
"cdrecord -scanbus".

My question: Do you have plan to add "device letter" information on
"cdrecord -scanbus"? (I am not requesting a new feature because I know
cdrecord is open source and I should not tell others what he/she
should do.) I want to contribute it by myself but I don't know how to
map USB/Firewire CD-RW drive's driver letter (e.g. E:) to SCSI
bus/target/lun because USB/Firewire CD-RW has nothing to do with SCSI.
