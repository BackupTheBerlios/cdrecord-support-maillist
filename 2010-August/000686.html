"mwesten" <mwesten at verizon.net> wrote:

> Hi,
>
> I'm using Cdrtools-3.0 (on RIPLinux 10.3) to write DVDs.  DVD+Rs are written fine, but I have some DVD+RWs which were previously written by a DVR machine that I will want to rewrite.  The first one of those that I tried was only half-full (~2GB), and when I attempted to record a ~3GB iso, cdrecord said the disc was too small.  That disc was closed, so I assumed it would just overwrite it from the beginning.  I thought DVD+RWs did not need blanking, but I did blank=fast on it anyway.  After that, I happened to eject the disc before attempting another write.  I reinserted it and tried writing, but got the same error (too small).  I did blank=fast again, and then issued the write command again, but this time without ejecting the disc.  It did write, but apparently did not complete successfully, and the disc doesn't work.  It took 300 seconds to fixate, and then the drive hung with the light blinking.  After rebooting, I found that the toc is wrong, and some files give an Input/Output error when I try to read them.

The whole DVD+ system is non-standard and a quality desaster.
There is nothing like the round robin test that the DVD Forum does for DVD-
media.

In special writing DVD+RW from different drives causes many compatibility 
problems. You may try to run the DVD+RW blank "emulation" cdrecord blank=all
several times or "cdrecord -format -force" if that works.
