This is a write error after just filling up the RAM in the drive, so I assume 
no previous successful media write before.

What you should do is to call

cdrecord -atip 

to get the media manufacturer and retry with original Verbatim media and to call

cdrecord -minfo 

before writing to get the apparen media state before starting to write.
