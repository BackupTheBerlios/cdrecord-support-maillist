	Gentoo's cdrtools package has a "theological" issue with the
auto-detection of ACL.  The Gentoo Devs would like to open a bug asking
for manual control of if ACL should be used by cdrtools at compile time.

	The original issue was from someone who had ACL installed, installed
cdrtools, then uninstalled ACL.  Because they were unaware of the
dependency, cdrtools stopped working.
	Gentoo handles these optional dependencies explicitly via "use-flags"
(of which "acl" is one).  These usually map to --enable-acl or
--disable-acl configure options.
	cdrtools's automatic detection confuses this system.  Normally if the
"acl" use-flag was on, the package system would know that cdrtools had
ACL as a dependency.  If it was off it does not know if the dependency.
 They do not want to mimic/parallel cdrtools auto-detection of ACL.  The
also are afraid of modifying cdrtools's makefile/configure system for
fear of loosing the ability to report bugs because of said modifications.
	So it appears they want the ability to manual control if cdrtools uses
ACL or not at compile time.


	The Gentoo bug (and all the lively discussions behind this request,
including dissenting opinions) can be found at:

	https://bugs.gentoo.org/show_bug.cgi?id=198779
