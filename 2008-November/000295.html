Just some comments and a renewed request to troubleshoot 'path-list'
problems:

1. an attempt last year to run the following pipeline under 'sh.exe'
    on Win2k with 'HP 8200' burner hardware resulted in a hung system
    requiring reboot:

    mkisofs -v -V dvr1of2 -J -r . | cdrecord dev=2,0,0 -tao -data -

2. pathlist file must be in the root of the source files when
    not running under a Cygwin shell (bash.exe), otherwise
    mkisofs aborts with "invalid node" errors.  In other words,
    it is not possible to specify a pathname for the pathlist
    file other than e.g. foobar.lst

    Also, blank lines result in an "invalid node" error, even
    at the end of file.

3. running under 'bash.exe', mkisofs '-path-list' parsing fails with
    "invalid node" errors when absolute or relative directory-to-
    pathname-mapping specifies a Novell share but succeeds when
    specifying a local drive; specifying a location on the Novell
    share without '-path-list' has always worked however.

Items (2) and (3) were previously reported to this list.

Michael
