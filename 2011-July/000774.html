> I am able to reproduce the problem, or something much like it,
> on my Linux box. My system uses a firewall that blocks all incoming
> connections except those related to established connections.
>
> Ftp is a strange protocol that historically requires two connections
> (in "active" mode). The first one, on port 21, is established by the
> client. The ftp server then initiates a new connection on its port 20
> for the data transfers. This latter connection will be blocked by a
> firewall that disallows any new connections initiated from the outside.
>
> To get around this, invoke passive mode which uses only the first
> connection by entering "passive" on the ftp command line:
>
>
> [mshell]: ftp ftp.berlios.de
> Connected to snake.berlios.de.
> 220 ProFTPD 1.3.3d Server (BerliOS) [::ffff:195.37.77.141]
> Name (ftp.berlios.de:mshell): anonymous
> 331 Anonymous login ok, send your complete email address as your password
> Password:
> 230-Hello ::ffff:208.100.228.166, there are 2 (max 40) users logged in at the moment.
>   Local time is: Sat Jul 09 11:09:00 2011
>
>   Please use your email address as password and NOT silly words
>   like "mozilla@" or "WWWuser@", as some Web browsers do!
>
>   If you have any questions concerning this ftp archive or would
>   like to announce your uploads, please send a mail to the ftp-admin
>   of this server: berlios-admin at berlios.de
>
>   All transfers are logged. If you don't like this, disconnect now.
> 230 Anonymous access granted, restrictions apply
> Remote system type is UNIX.
> Using binary mode to transfer files.
> ftp>  passive
> Passive mode on.
> ftp>  ls
> 227 Entering Passive Mode (195,37,77,141,209,222).
> 150 Opening ASCII mode data connection for file list
> .
> .
> ----
>
>
> Some Windows clients use different names for the passive command, and
> some do not support passive mode at all:
>
> http://www.windowskb.com/Uwe/Forum.aspx/windowsxp/264689/How-to-configure-ftp-exe-to-use-passive-mode-PASV
>
> If your ftp client does not support passive mode, then you will need to get
> one that does (there are many). This is yet another reason to dislike
> Microsoft products.
