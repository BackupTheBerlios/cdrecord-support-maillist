> If you did call cdda2wav as root, then you are a victim of the
> problem, that cdda2wav does not know whether it gained fine grained
> privileges from facps or suid root or whether it has been called by
> root. It always drops special local file privileges making root a
> normal non-privileged user.
> 
> If you did not call cdda2wav as root and cdda2wav did gain it's
> privileges from suid root, you need to make a bug report against the
> Linux kernel as cdda2wav restores the real user id to be the
> effective user id before creating the files.
