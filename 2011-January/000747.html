A standard CD starts at sector 0 == +2 seconds. One additional second is not 
called a hidden track.

> "-o 75" and yields asmaller file than without "-o", I checked that
> there is a noticeable difference when the original is played in the CD

Read the man pages....unless you tell cdda2wav to ignore hidden tracks, it
extracts them un bulk mode. You of course need to tell cdrecord to use the 
meta data information from cdda2wav (see man pages).

I see no reason why there should be a difference.....
