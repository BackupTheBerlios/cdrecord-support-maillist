Anyway, are there any problems with SATA drives and FreeDOS? Any Microsoft
OS boot disks aren't going to have SATA support prior to Win2000. If the
computer is using the motherboard SATA hardware, the BIOS should support
that, but I don't know if there are any other potential issues here with
a modern BIOS SATA API presented to an older OS (e.g., 200GB+ drives).

I've never used Wine yet myself, but it is an interesting option.
Looking at the FAQ:

http://wiki.winehq.org/FAQ

It does NOT require Windows. It is an independent Unix reimplementation
of the Windows API! It is is a set of Unix libraries that map Windows
system calls to those of the Unix host system. So, a Windows binary will
think it is running under Windows when in fact it is actually calling a
Unix library that maps the calls to the kernel and X. The wine source
tar ball download is only about 11MB, so if the setup is not too much,
it might be fun to try:

http://sourceforge.net/projects/wine/files/

For those of you that know how to configure PATH, /etc/ld.so.conf and
/etc/man.conf could install the whole thing in /opt/wine, add the
/opt/wine/bin, /opt/wine/lib and /opt/win/man paths and try it out. If
you don't like it, tis easy enough to yank out without touching the
rest of the system. Note cdrtools likes to use a similar approach with
/opt/schily.

Beware that any Windows malware running under wine can touch the Unix
filesystem. For this reason, run only trusted applications, do not run
wine as root, and perhaps use a different user account. However, you
might have to run under root for a flasher utility. Anyway, although
Windows malware might be able to alter or delete files, it probably
would not be able to "infect" the Unix system because the files it
would have to target would be totally different from Windows.
