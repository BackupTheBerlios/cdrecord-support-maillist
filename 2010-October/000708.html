Hi,

I've written a debian packaging from scratch (for Ubuntu Linux
Maverick 10.10) of the cdrtools 3.00 stable release:
https://launchpad.net/~eshattow/+archive/cdrtools-stable

There is also a modern packaging of 3.00 stable release:
https://launchpad.net/~brandonsnider/+archive/cdrtools

The difference there is I make no attempt to integrate cdrtools with
cdrkit / dvd+rw-tools in my packaging effort; All executable files are
renamed with a suffix "-cdrtools" on the name:
/usr/bin/cdrecord
becomes
/usr/bin/cdrecord-cdrtools

and this is done with all installed executables.  Any static libraries
in /usr/lib are moved to /usr/lib/cdrtools ; similar with installed
header files moved to /usr/include/cdrtools

Jeorg, is this use of renaming acceptable?
