"Bruce Hill, Jr." <servants at cotse.net> wrote:

> The Asus DRW-2014L1T SATA drive recently purchased will read
> CD/VCD/DVD, but will not write to them using cdrecord, cdrdao, or
> growisofs. The latest version of all three apps are installed.
>
> I have tried many different kernels, and burned using K3B, also,
> with basically the same results.
>
> The same computer has an IDE DVD+/-RW, which burns successfully with
> the following commands. I've tried 4 different types/brands of media
> with the same results.

There are three possible reasons for your problem:

1)	HALD distrurbes CD/DVD writing. In this case just kill hald and verify
	that it is really not running while you try to write a CD


2)	The Linux kernel did bastardize the SCSI commsnd that cdrecord tries to
	send.

	There is a simple way to check for this, just boot an OpenSolaris life 
	CD (SchilliX ftp://ftp.berlios.de/pub/schillix) and try to write a file
	from Solaris

3)	Your drive has a firmware bug that prevents writing on SAO mode.
	In this case follow the instructions in the error message and specify
	the -raw96r option.
