is not lying about the cause, then the drive is unable to deal with the media.

----> Sense Code: 0x0C Qual 0x00 (write error)

Plasmon stampers are frequently used by cheap no-name manufacturers.
Could you try a Verbatim medium?

I would call the visibility of these messages a kernel bug.

Cdrecord sends RAW SCSI commands and SCSI is a complex protocol that 
sometimes results in "failing" commands as a result of a test. Only the sender
of the commands (cdrecord) may know whether it is expected that a specific
command fails, so having this is the kernel messages is wrong as there
could be millions of similar messages.
