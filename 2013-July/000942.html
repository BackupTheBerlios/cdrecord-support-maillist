smake release 1.2.3 was published on January 30, but the last change on cpp was 
on January 4th already, so I would guess that there is no change that was in 
the "schily" tree.

The command is:

OBJ/i386-sunos5-cc/cpp  :       cpp.o cpy.o  
        $(LDCC) $(_LD_OUTPUT_OPTION) $(POFILES) $(LDFLAGS) $(LDLIBS) 

in the makefile.

POFILES  =      $(CFILES:%.c=$(ARCHDIR)/%.$o)   $(C++POFILES)

LDFLAGS  =      $(LDOPTS) $(LDOPTX)  

LDOPTS  =       $(LIBS_PATH) $(LDPATH) $(RUNPATH)  

LIBS_PATH  =    -L$(OLIBSDIR) $(LIBS_PATH_STATIC)

I see no chance for -y to slip in here....
