Mark Knecht <markknecht at gmail.com> wrote:

> Hi,
>    What is the best way to determine what media works in a specific
> drive? I'm trying -checkdrive but that's not very clear. Is there a
> command which produces output that might say DVD-R supported, DVD+RW
> not supported for writing, supported for reading, etc.?

The best way is to write a media and later check the write quality.
For CDs, things are relatively easy as many drives support the read C2 pointers
command, even though C2 errors are uncorrectable high level errors at the upper
audio level (data sectors sit on top of audio sectors).

Unfortunately the command set for doing the needed checks for DVDs is known only 
for a few drives. For this reason, the only way to check in most cases is to 
start:
	readcd meshpoints=1000

catch the output and visualize the result using e.g. gnu plot. If you see slow 
downs in some areas, the write quality was bad. If you have one of the last 
drives made by Plextor and installed a hacked firmware, you may run Pisum and 
Pi8 checks.

> BACKGROUND:
>
>    I did a backup to DVD sometime ago. I have a DVD drive in the
> machine I backed up but I had a hard time writing the DVD so I moved
> all the data to a second machine and did the backup there. The DVD is
> readable on the machine that made it, and also on a 3rd machine, but
> it's not readable (so far) on the machine where I need to now restore
> the data. Is this because the drive in the machine is bad, or is it
> because the drive is not compatible with the media I wrote the backup
> on? (DVD-R)

Sure that it was not a DVD+?

The DVD Forum standards are carefully checked by doing "round robin" read/write
tests with all media using all writers and readers. If there is a problem, the 
manufacturers are informed immediately. 

This does not apply to DVD+ media.

If you like good readability, I recommend Verbatim media.
