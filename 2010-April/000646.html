Eric Shattow <lucent at gmail.com> wrote:

> A modern feature of audio CD ripper is sample offset correction. I
> noticed there is a setting for frame offset, this is not what I mean.
>
> The softwares cdparanoia (http://xiph.org/paranoia) and cued

Are you sure that cdparanoia has this feature? Cdparanoia is based
on a very old versioon of cdda2wav and thus does not implement most of the
features found in recent cdda2wav versions. Cdparanoia is also non-prtable
and Linux only while cdda2wav works on more than 30 different OS platforms.

> (http://gna.org/projects/cued) have the capability to apply sample
> offset correction, and under/over read the media on supported drives
> to "recover" these normally missed samples.

It is most unlikely that this program will help you. It is based on libcdio
and libcdio was created from an outdated version of cdda2wav and is even 
in conflict with the copyright law as the license was changed to GPLv3 without 
the permission of the Author.. 

Knowing the code from libcdio, it seems to be very unlikely that libcdio is 
portable software. ....just checked: it is far away from being portable.

> Please note this feature request is not intended to spark a discussion
> on the merits of AccurateRip(TM) compatibility, or "what is zero
> (reference) anyways?" It is just a feature that I would find useful if
> it were implemented. CDROM drive hardware varies a lot about which
> sample to begin reading from, and being able to compensate for this is
> important for my uses. I am surprised that a sample offset
> compensation read feature does not appear to be implemented already in
> cdrtools.

Well, the features in cdda2wav are oriented on importance and offset correction
is a task of low importance if you think of general audio extraction quality
and if you look at the ability to extrace meta data from or for audio CDs.
With general audio extraction quality, cdda2wav is one of the very best 
programs and e.g. better than cdparanoia. Cdda2wav reads many audio CDs 
without problems that cannot be read using cdparanoia at all.

The next planned feature for cdda2wav is the ability to detect stable incorrect 
audio data and after that, offset correction is planned.

Note that the typical audio read offset is less than 20ms and you will need to 
create a 20th or 50th stage copy in order to be able to notice the offset at 
all.
