This is not really true. There is typically better documentation for UNIX tools 
than for MS-WIN tools. This is because Steve Bourne created a nightly cron job 
in the mid 1970 when /usr/bin/ was added as a global wriable directory for 
locally developed applications that could be of interest for more than a single 
person.

Bourne removed all binaries that have been updated without updating the manpage 
also.

This is why UNIX programmers usualls write good documentation.

The documentation for cdrtools is also updated on a regular base and always 
tries to include useful feedback from users. 

I have the impression that the biggest problem with cdrtools is that many 
people don't read the documentation.

During the past months, the cdrtools documentation was heavily modified on order 
to make it easier to read. If you have proposals on how to further increase 
usability, you are of course welcome.....
