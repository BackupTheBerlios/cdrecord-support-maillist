  Mark,

Although I have yet to get around to learning the details of bootable
CD's, I do know a bit about the boot process on hard drives.

The problem is that a bootable disk must contain boot loader code, not
just files.

I think you are asking about how to make "El Torito" bootable disks:

http://tldp.org/HOWTO/Bootdisk-HOWTO/cd-roms.html
http://odin.himinbi.org/xp_cds/eltorito_extraction.html

That would involve the use of the mkisofs -b and possibly -hard-disk-boot
options - if you really do need to use mkisofs (see below).

The second URL above takes you through the process of acquiring the boot
code, but it does involve picking through a hexdump.

I do not understand this:

> In the past I've used mkisofs to rip iso images of Windows operating
> system disks (Win98, Me & XP) to do an install in VMWare-Workstation.
> I suspect my old command might not be appropriate anymore?

What I don't get is why mkisofs would be used if all you want to do is
duplicate the disks nor do I understand why so doing would ever result
in a bootable CD if the -b option was never invoked.

From the second URL above:

: Making a straight copy is easy enough, except the sand eats all discs here
: and I don't really want a disc anyhow. I just want to be able to make one.
:
: With linux this is pretty easy to pull off.
: dd if=/dev/scd0 of=isos/win_xp.iso
:
: will give me a iso image that I can verify is good by doing:
: mount -o loop win_xp.iso /mnt/winxp/
:
: When I get ready to burn it I just do:
: cdrecord -v speed=2 dev=0,0 ~will/isos/win_xp.iso
:
: This is all fine and good, but I would like to keep the key with the disc.
: My initial idea was just to mount the iso image and try writing into it.
: The structure of an iso though prevents this from working. A simple solution
: is just to copy the files out and make a new iso. The problem? The disc is
: bootable and it won't be if I just make a new image. I need to copy the boot
: image out and put it in the new one.


I am not totally sure about "would like to keep the key with the disc",  but if
I understand him correctly, he wants to add a little .txt file that holds the
registration key to the image, which is why he needs to reconstruct the iso
image from scratch.

So, do you really need to modify the images? If not, a simple dd:

dd if=/dev/scd0 of=win_xp.iso

will grab the complete image including the boot code.
