I cannot see any solution that preventy you from reading the man pages.

CD/DVD/BD writing is a complex process that has been made simple by cdrtools.
In order to allow all features that users may think of, there need to be  many 
options related to these features. Mkisofs e.g. implements 167 options, this is 
_much_ more than what "ls" offers. The man page for "ls" (36 options) has 24 
pages, the man page for mkisofs has 53 pages.

with a nice cookbook that mentions all typical boot methods and gives many 
examples. If you believe that this is not sufficient, please quote the related
