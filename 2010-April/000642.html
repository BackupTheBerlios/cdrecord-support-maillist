I don't think the man pages are necessarily the right place to put
things in the 21st century.

Just like the current blue links, how about some user-maintained Wiki page at

http://cdrecord.berlios.de/private/cdrecord.html

where we can maybe create a set of example commands that are more task
oriented instead of spec oriented?

For instance, when I look at the man page here:

http://cdrecord.berlios.de/private/man/cdrecord/mkisofs.8.html

I find an example:

     To create a CD with Rock  Ridge  extensions  of  the  source
     directory cd_dir:

     % mkisofs -o cd.iso -R cd_dir

but I don't know when or why to use Rock Ridge vs something else.
(Honestly, I don't!) And I don't think I should care what technology
is required to make my CD. I'm not interested. I just want a CD (DVD,
multi-media CD, etc...)

Now, I didn't think that command actually made a CD. I thought it made
an iso image that could later be burned to a CD. So maybe this is
something that could be explained more at the user task level than the
software operational level, edited and maintained over time sort of
like a lot of the Gentoo Wiki pages? I.e. - to make a bootable CD copy
do:

Step 1 - create an iso copy of the CD using this command
Step 2 - burn the iso image to a new, blank CD using this command

Just an idea.
