Yeah, it's a problem, but I have a hard time picking on the cdrecord
toolset developers and maintainers on that account. There's pleanty of
verbage in man mkisofs. The question gets to be whether the answer I'm
looking for is even in there and then whether all the verbage in the
world will ever get through to someone (me!) who doesn't want to
become an expert like Joerg?

I don't want to be an expert.
I don't want to read source code.
I don't want to search around my machine looking for files that might
contain answers but actually don't.
It's not good use of my time.

As a user I want a really specific, terse example of the sort of job
I'm trying to do so that I can apply it and get the right results.

As an example of how time goes by, I messed around for 2-3 hours
before I wrote the first message to the list 2 days ago. Since then
I've spent maybe 15-20 minutes writing responses, I've spent NO more
time using mkisofs, and I still don't have a bootable iso image of the
Win7 DVD on my system. What did I do? I booted the VM from the DVD
itself and moved on. mkisofs failed me, but not because it's not
capable. It failed because it's not capable of dealing with Type A,
impatient, time-limited distracted old idiots like me! ;-) (And I'm
not sure it should be!)
