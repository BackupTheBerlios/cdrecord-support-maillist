It's a bit sad because I actually like to write documentation. ;) And, at
least in my case, the documentation is very closely tied to the software
development - to the point where I sometimes find that during the course
of writing the documentation I come to dislike a part of the overall
architecture which showed up only when having to explain it, and as a
result, the code is changed and improved.

Often, I will write the documentation *before* coding as the docs then
serve as the reference for what the application is supposed to do.

The above said, I'm not a fan of Knuth's literate programming - doing
a task and explaining it, although closely related, are very different
things.

I do like your Summer of Documentation idea. :)
