Hello,
The alpha versions of mkisofs are producing ISO images with a certain incompatibility (against ECMA-119 standard): 
 
 
 Data in "Creation Date and Time" field(s) of "Volume descriptor" is incorrect (it contains spaces). 
 
 
The current mkisofs 3.00 stable version is correctly building ISO images, including the mentioned fields.
 
Version 3.00 under Windows and 3.01a02 under SalixOS both work correctly (the data fields mentioned are correct).
 
Version 3.01a07, both under Windows and under SalixOS, produces an "incompatible" ISO image (the data fields mentioned are INcorrect).
 
Using the same directories and files as origin in every attempt, I built the same ISO images under Windows and under Salix, using the same command line arguments and options, each time with different versions of mkisofs.
 
I believe the incompatibility started in version 3.01a05 (if I am not mistaken, version 3.01a04 was OK regarding these data fields), but please don't just take my word.
 
Versions 3.01a06 and 3.01a07 are not complying with the ECMA-119 standard for these data fields.
 
I can provide more info if necessary.
 
Thank you,
Ady.
