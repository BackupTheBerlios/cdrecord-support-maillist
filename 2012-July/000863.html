I would guess that your cue file is not correct.

Exact Audio Copy is known for creating cue files that do not follow the CUE 
file documentation from the "inventor" of this file format (CDRWIN).

The background is that the CUE format is extremely limited - this is why 
cdrecord and cdda2wav by default use the *.inf files.

Anyway: as you do not give any information, I don't know how to help.
