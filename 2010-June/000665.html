> I do have permission to /dev/sg0 (a CD device) but not to /dev/sg1
> (SATA hard disk).  That is what I want as an administrator / user.
>
> What prevents cdrtools "Auto Target" from ignoring /dev/sg? on Linux
> when permissions are refused, and processing only /dev/sg? that it
> does have permissions to use? Is this unique to Linux?

Didn't I explain this already?

Being able to open a /dev/ entry is no longer sufficient in Linux.
There was an incompatible Linux Kernel interface change ragarding to SCSI
in September 2004 and after that change, you need to be root _after_
opening the devices in order to send all needed SCSI commands.

Once there is a Linux distro that supports to give cdrecord fine grained 
privileges _witout_ the need to make cdrecord run under root, ther eis a chance
to enhance cdrecord to support this method. Currently there is none of the
wide spread Linux distros that would allow to do this. As a resuld, cdrecord,
cdda2wav, readcd and rscsi need to be installed suid root.
