> Which cdrtools executables are safe to install suid root?   Is
> cdrecord the only one?
>
> Does i.e. cdda2wav require suid root?

If you call "smake install", andif you are root, every program that needs root
privileges will be installed suid root.


> My concern is that auto target support breaks needlessly if we do not
> have permission to all /dev/sg? devices.

This is a concept from the Linux kernel folks. I cannot change it.

BTW: you would not be happy with being able to open a device but later 
cannot send the SCSI commands you need to send tothe device. This is where
you need root privieges anyway.
