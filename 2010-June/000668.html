The true story is this:

With every Linux version before aprox. spring 2004, you needed to be 
root _and_ able to open the device files READ/WRITE in order to be able
to send SCSI commands at all. Then a kernel novice (Douglas Gilbert)
enhanced a driver wihout thinking about the results. As a result, SCSI 
commands could be send with a ReadOnly file descriptor from the device.
As many related driver entries may be opened ReadOnly by everyone,
everyone could send any SCSI command.

Instead of fixing this bug and leaving the kernel interfaces untouched, 
Linus Torvalds introduced an unneeded Kernel interface change without 
correctly (or even completely) fixing the problem. Many people in LKML 
protested but Linus ignored everything.
