Michael Shell <list1 at michaelshell.org> wrote:

> > With every Linux version before aprox. spring 2004, you needed to be 
> > root _and_ able to open the device files READ/WRITE in order to be able
> > to send SCSI commands at all. Then a kernel novice (Douglas Gilbert)
> > enhanced a driver wihout thinking about the results. As a result, SCSI 
> > commands could be send with a ReadOnly file descriptor from the device.
> > As many related driver entries may be opened ReadOnly by everyone,
> > everyone could send any SCSI command.
>
>
> Then let me ask this: Prior to 2004, was it not true that a user did not
> have to run cdrecord as root for it to be able to access the drive,
> or was that only the case after Douglas Gilbert's 2004 changes?

Before that date, it was absolutely impossible to send any SCSI command
unless you opened /dev/sg* RW as root.

> Are you saying that with the Gilbert changes, even though the file
> descriptor was opened as read only, that any SCSI command could be sent,
> which means that the drive could be ordered to do just about anything,
> including writes? (meaning that the system never really was secure
> after that even under normal user access)

Correct, after the changes had been included, writes to the drive have been 
possible for non-root users with a read-only file descriptor.
