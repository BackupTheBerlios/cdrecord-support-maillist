 I think Joerg is referring to:

http://www.faqs.org/docs/bashman/bashref_76.html

specifically "28. Subshells spawned to execute command substitutions
inherit the value of the `-e' option from the parent shell. When not
in POSIX mode, Bash clears the `-e' option in such subshells."

It is possible to enable bash's POSIX mode at run time.

But, I can see some reasons why POSIX mode of bash can be such a
problem with existing systems:

"7. The POSIX 1003.2 startup files are executed ($ENV) rather than the
    normal Bash files."

Although I can't say the /etc/profile and .bashrc approach is totally
sane.

Which UNIXes use bash in POSIX mode by default?

Or, is there a bug with bash in that it does not follow its own
documentation? That most certainly would be a bug.

Why is that && isn't what is required rather than & to be able
to see an exit 1 status?
