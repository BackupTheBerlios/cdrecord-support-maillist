You are on Linux and this is a well known bash bug :-(

	bash sometimes ignores the "-e" option.

It is unfortunately even impossible to find a workaround for _this_ bug.
Smake already includes workarounds for _other_ bash bugs....

One way to fix the problem is to fetch the schily source consolidation

ftp://ftp.berlios.de/pub/schily

install "sh" as "/bin/bosh" and edit smake/make.c to call "/bin/bosh"
instead of "/bin/sh".
