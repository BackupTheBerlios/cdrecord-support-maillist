There is an interesting article by an IT guy related to this concerning

One of the great things about Unix is the multiple levels of
configuration (for applications that are done correctly). There's
a system wide setting in /etc and per-user settings in .files
in the user's home directory. This way, the machine can be set
by root with a default configuration for all users, and each
user can override as needed (without needing root privileges).

It is also possible to have /home/mydir/bin and reconfigure
PATH to allow for user executables.

The reason for the concept of file ownership is security. It
can be a pain to get used to, but in the long run it can
save time by not having to deal so much with broken or
compromised systems.

Unix guru Henry Spencer said, "Those who do not understand
Unix are condemned to reinvent it, poorly." I think Microsoft
has been heading in that direction for some time.
