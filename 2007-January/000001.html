Skunk Worx <skunkworx at verizon.net> wrote:

> Hi,
>
> I use FC6 (Fedora Core 6) to master DVDs and have noticed if I add the 
> -J option to the mkisofs command line the read performance on the final 
> dvd is severely degraded.
>
> For example mastering a DVD image with RockRidge and ISO9660 alone 
> yields about 200 mb./minute read performance on my media.
>
> Adding the Joliet option to mkisofs command line degrades the read 
> performance on the final dvd to about 75 mb./minute.

Why do you believe that this is caused by mkisofs?


> Is this expected behavior? I used the same media and drive (dvd-rw) and 
> blanked with /dev/zero between cycles then duplicated the test twice to 
> be sure.
>
> $ mkisofs -version
> mkisofs 2.01 (cpu-pc-linux-gnu)

This is an extremely old version (~ 2.5 years), it is not even clear whether 
it has been crippled by RedHat.,....

mkisofs does not behave different when creating Joliet...

I recommend to test again on a modern OS like Solaris and 
I recommend to switch to a recent _original_ version of mkisofs.
