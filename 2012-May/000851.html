Lucas Levrel <llevrel at yahoo.fr> wrote:

> Thanks all for the valuable info. The official specs are a bit tough (as all specs are), but it'll help me understand the CD layout anyway.
>
> I didn't know isoinfo; plus I thought ISO9660 referred only to the filesystem, so I wouldn't have discovered "isoinfo -d" without your help.
>
> > Note that most boots today use the no-emulation boot with an apparent boot size 
> > of 2 kB. As the code in the first 2k will load the rest from the directly 
> > following sector numbers and as usually there is no file name for that object, 
> > you don't know how much to extract.
>
> Can't you seek until you find the first sector of the filesystem, and extract everything up to there?

You can only make some guesses as the boot code may be anywhere.
If you verify that the boot start sector is identical to the start sector of a 
ISO-9660 file (see isoinfo), then cou can assume that you know the real file 
size.
