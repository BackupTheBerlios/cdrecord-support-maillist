Sebastien TOPIN <sebastien.topin at waaves.com> wrote:

> I am trying to use "cdrecord + mkisofs" to add a session on a non  
> finalized CD-R that already contains data burned on Windows. I use ISO  
> 9660 level 1 + relaxed Joliet options on Windows to support longer  
> names.
>
> Now I can add a new session to my CD on Mac OS X (10.5.8) also  
> supporting long file names using Joliet. The only problem is that it  
> truncates all file names from previous sessions to strict ISO 9660  
> "8+3" characters.

This is an unfortunate result of using the non-standard Joliet.
While there is a 100% 1:1 relation between ISO-9660 file names and the 
Rock Ridge extensions, such a relation does not exist between ISO-9660
and Joliet. For this reason, it is not possible to keep any file names that
only exist in Joliet.

> I also tried to add the following option but that did not change  
> anything:
> -allow-lowercase
> -relaxed-filenames
> - r or -R (that does not work since it claims that the previous  
> session does not contain Rock ridge option)
>
> If I use ISO 9660 level 2 on Windows side to burn the first session,  
> that solves the problem up to 31 characters but I would need to be  
> able to use ISO 9660 level 1 for compatibility reasons on Windows side.
>
> Any ideas how I can solve this problem?

Use Rock Ridge extensions starting with the first session.
