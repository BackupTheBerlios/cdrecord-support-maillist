Mark Knecht <markknecht at gmail.com> wrote:

> Am I supposed to use mkisofs or something else BEFORE writing data
> backups to a DVD+R disc? Is there a good web page on doing this with
> cdrtools? All the pages I'm finding are for using GUI frontends and I
> want to learn to do this from the command line.

Did you look at the EXAMPLES sections in the various man pages?

http://cdrecord.berlios.de/private/man/cdrecord/index.html


> I've never used cdrecord from the command line to do backups so I
> thought as a test I would try writing a large tar file to a DVD+R disc
> as data. The command I used was about as simple as possible and
> probably not right:
>
> cdrecord dev=1001,0,0 backup.tar.bz2

This is possible but not really recommended as such a medium will not mount
because there is no file system on it.


> The command seemed to have executed so I ejected the disc and took it
> to another system where I tried:
>
> firefly ~ # mount -t iso9660 /dev/dvdrw /mnt/tmpMnt/
> mount: block device /dev/sr0 is write-protected, mounting read-only
> mount: wrong fs type, bad option, bad superblock on /dev/sr0,
>        missing codepage or helper program, or other error
>        In some cases useful info is found in syslog - try
>        dmesg | tail  or so

See above, there is no filesystem ;-)

You may like to look at the option -stream-media-size from mkisofs.

If you have no problem to remember that such a medium contains a tar
archive, you may continue.
