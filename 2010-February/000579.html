Cia Watson <ciamarie at my180.net> wrote:

> If I may, I'd like to make the suggestion that if there is an error in
> writing (like a memory error, I think I can give output if you want)
> and 0 bytes get written, that it not fixate the disk to which nothing
> was written?

This is what cdrecord does. In case that it is 100% clear that nothing was 
written, cdrecord just exits.

> I had that happen twice today, once because I forgot to tell it not to
> write at a higher speed, which always results in a write error,
> before it even starts to write; and it says 0 bytes written and then it
> fixates the disk. It is highly annoying since I have a lot more
> coasters than I need at this point. Thanks. 

0 bytes written does not also gran that other areas of the medium have been 
untouched.

If you cannot write at the default speed, you should consider buying better 
blanks or check whether there is a firmware upgrade for your writer that might
help.
