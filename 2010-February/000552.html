Sebastien TOPIN <sebastien.topin at waaves.com> wrote:

> Thanks for your answer.
>
> I understand that Joliet is not standard but to my knowledge there is  
> no Rock Ridge support on Windows platform and I need to be compatible  
> with CDs burned on Windows.

Rock Ridge is a standard compliant extension to ISO-9660 and there is no
problem to crerate RR enhanced CDs using mkisofs on MS-WIN. Note that
the RR directory information is sored in a knapsack to the related ISO-9660
directors entry.


> Does that mean that there is no solution to my problem using ISO 9660  
> level 1?
> Should I try to find parameters that give the same Joliet result on  
> Mac and Windows?

The problem is that Joliet is a completely separate filesystem structure 
and for this reason it is in general not possible to find out the relation 
between a ISO-9660 file name and a Joliet file name.
