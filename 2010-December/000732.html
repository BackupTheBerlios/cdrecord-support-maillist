Infra <msg at waste.org> wrote:

> Installed fbk driver on sol 2.6/sparc, postinstall completed w/out error
> despite warning in README. Ran drvconfig and devlinks. /dev/fkb0 and
> /devices/... files exist. Also ran modload drv/fbk. No errors anywhere.
>
> # mount -r -F fbk /dev/fbk0:/server2.a0/bigsys/usr2/usr2-bu/c0t2d0s1.dd
> /mnt
> MOUNT FBK: mount -r /dev/fbk0:/server2.a0/bigsys/usr2/usr2-bu/c0t2d0s1.dd
> /mnt
> mount -F ufs -m -r /dev/fbk0 /mnt
> mount: /dev/fbk0 no such device
> mount: mount exited with 32
>
> Really don't want to reboot production machine. Anything I can do?

There is no need to reboot.


If there is no /dev/fbk0, the postinstall did not work correctly. Run 

drvconfig
devlinks

this should create it.

If this is no ufs, then you need to mount the image by hand later using:

mount -F something  /dev/fbk0 /mnt

Theassociation between /dev/fbk0 and the file is removed after the last close.
