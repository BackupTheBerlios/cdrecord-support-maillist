Greetings:

Does this issue occur in current release mkisofs on Win32? (I quote from my
notes):

> Notes on 'mkisofs':
> 
> 1. Use forward slashes in '-path-list' file entries.
> 
> 2. pathlist file must be in the root of the source files when
>    not running under a Cygwin shell (bash.exe), otherwise
>    mkisofs aborts with "invalid node" errors.  In other words,
>    it is not possible to specify a pathname for the pathlist
>    file other than e.g. foobar.lst
> 
>    Also, blank lines result in an "invalid node" error, even
>    at the end of file.
> 
> 3. example use of pathlist for recursive directory tree iso:
>    mkisofs -J -r -o foobar.iso -graft-points -path-list foobar.lst
> 
>    Contents of foobar.lst:
> 
>       dir1/=dir1
> 
>    command is executed in the root of the source files (which
>    include 'dir1').  In pathlist, left value=name on cd,
>    right value=source.
> 
>    Note that when not using '-path-list', the format of the
>    graft point in the mkisofs command line is the same as in
>    the pathlist file.

Happens for me on versions 2.01, 1.11a12 and 1.15a12 compiled for
cygwin. The blank line issue is trivial, but the -path-list file
not located in the source root problem is troublesome.

Regards,

Michael
