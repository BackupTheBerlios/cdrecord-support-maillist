Built 2.01.01a37 (default download from 'alpha', labeled 'beta') on
Cygwin (dll 1.3.22), Win2k Server SP3, GCC 3.2:

1. libsiconv build fails:
      sic_nls.c:44:9: missing binary operator before '('
    -- commented out lines 44 -> 48

2. tested 'mkisofs':
      ./mkisofs -J -r -o foobar.iso -graft-points -path-list foobar.lst
      ./mkisofs: Warning: Cannot add inode hints with -no-cache-inodes.
      '.mkisofs: No such file or directory. Invalid node - 'dir1
    (NOTE: superfluous single quotes, not present in output from older
     builds of mkisofs)

3. determined that "Invalid node" error is due to presence of 'CR LF'
    EOL delimiters vs. LF-only delimiters in pathlist file.
    NOTE: this is also the reason for the error seen in earlier builds
    of mkisofs.

I am reporting this FWIW and apologize for not studying the code and
submitting a patch (due to time constraints).  I feel that being able
to use cdrtools such as mkisofs and cdrecord on Win32 without needing
to install the cygwin environment is useful; one need only move
cygwin1.dll and the tool binary to a target machine. In these cases
it would be necessary to permit creating and editing control files
such as 'pathlist' in native editors (or even 'copy con filename')
so CR/LF should be permitted.
