Udo Büdel <ubuedel at gmx.de> wrote:

> JOYDEEP wrote:
>
> > During the speed selection function I like to implement two tests.
> > 1> It check the maximum writer speed of the drive
> > 2> It check the maximum allowable write speed of the disk
>
> Basically insert a media and run 'cdrecord -prcap'
> Look for Maximum write speed: and Write speed's.
>
> This report drive information.
> However there are strange firmwares.
> Despide this writer information, a writer may write at different write
> speeds.

This does not work for all drives and you need to check the whole part
from:

  Maximum read  speed:  8468 kB/s (CD  48x, DVD  6x, BD  1x)
  Current read  speed: 16620 kB/s (CD  94x, DVD 12x, BD  3x)
  Maximum write speed:  8310 kB/s (CD  47x, DVD  6x, BD  1x)
  Current write speed:  8310 kB/s (CD  47x, DVD  6x, BD  1x)
  Rotational control selected: CLV/PCAV
  Buffer size in KB: 2048
  Copy management revision supported: 1
  Number of supported write speeds: 2
  Write speed # 0:     0 kB/s CLV/PCAV (CD   0x, DVD  0x, BD  0x)
  Write speed # 1:     0 kB/s CLV/PCAV (CD   0x, DVD  0x, BD  0x)

Current performance according to MMC get performance:

Maximum performance according to MMC get performance:
End LBA:      158644
Read Speed:    11080 == 8x DVD
Write Speed:    8310 == 6x DVD

End LBA:      158644
Read Speed:    11080 == 8x DVD
Write Speed:    5540 == 4x DVD

The first block is MMC-2, the rest is MMC-4

I am thinking about adding the max write speed to the cdrecord -minfo
output in future, but even this may not be always correct due to firmware bugs
in the drives.
